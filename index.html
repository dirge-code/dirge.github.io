01 · CAPABILITIES

Engineered for Speed & Efficiency

dirge is built from the ground up in Rust with a focus on minimal resource usage and maximum utility. Every feature is designed to get out of your way.

01

Terminal-First TUI

Built with ratatui + crossterm. Stream tokens in real-time, manage sessions, browse conversation trees, and toggle info panels — all in your terminal.

02

25 Built-in Tools

Read, write, edit, grep, glob, bash, webfetch, websearch, memory, task delegation, semantic code analysis, LSP integration, and more.

03

7+ LLM Providers

OpenAI, Anthropic, Gemini, DeepSeek, Ollama, OpenRouter, GLM — plus any OpenAI-compatible endpoint. Swap models with /model.

04

Plugin System

Extend with Janet scripts. Hooks for every lifecycle event: prompt, tool calls, responses, sessions. 17 example plugins included.

05

MCP & ACP Support

Connect Model Context Protocol servers for extra tools. Agent Communication Protocol for editor integration (Zed, VSCode roadmap).

06

Session Management

Save, resume, fork, and branch conversations. Auto-compaction keeps context within limits. Tree view for visualizing conversation forks.

07

Semantic Code Tools

Tree-sitter AST analysis for 10 languages: list symbols, find callers, find callees, get symbol body, find definitions.

08

LSP Integration

9 language servers auto-attach on file edit. Diagnostics injected into tool output — catch errors as the agent works.

09

Permission System

4 security modes: standard, restrictive, accept-all, yolo. Pattern-based allow/deny rules. Session allowlists. Doom-loop detection.

02 · GET STARTED

Install in Seconds

Build from source with Cargo. Set your API key and you're ready to code.

▸ Terminal — install
# Install with all features
$ cargo install --git https://github.com/yogthos/dirge

# Or with specific features
$ cargo install --git https://github.com/yogthos/dirge \
    --features semantic,lsp,mcp
▸ Terminal — build from source
# Clone and build
$ git clone https://github.com/yogthos/dirge
$ cd dirge

# Release build (recommended)
$ cargo build --release --features semantic,lsp,mcp
$ ./target/release/dirge
▸ Terminal — configure API key
# Set your API key (provider auto-detected)
$ export ANTHROPIC_API_KEY=sk-ant-...
$ export OPENAI_API_KEY=sk-...
$ export DEEPSEEK_API_KEY=sk-...

# Or configure in ~/.config/dirge/config.json
$ dirge --api-key sk-...

06 · CONFIGURATION

Declarative Config

Configure providers, permissions, themes, and MCP servers in ~/.config/dirge/config.json.

▸ ~/.config/dirge/config.json
{
  "provider": "openrouter",
  "model": "deepseek/deepseek-v4-flash",
  "theme": "phosphor",
  "permissions": {
    "mode": "standard",
    "rules": [
      { "pattern": "git checkout *", "action": "allow" },
      { "pattern": "rm -rf **", "action": "deny" }
    ]
  },
  "mcp_servers": [
    {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-server-filesystem"]
    }
  ]
}
07 · USAGE

Multiple Modes

dirge adapts to how you work — interactive TUI, one-shot print, headless loop, or editor integration via ACP.

▸ Terminal — usage examples
# Interactive TUI (default)
$ dirge

# One-shot: run and print to stdout
$ dirge -p "explain this codebase"

# Headless iterative loop
$ dirge --loop "refactor auth module"

# Resume previous session
$ dirge -c

# Editor integration (ACP)
$ dirge --acp

# Restrictive mode (prompt for every tool)
$ dirge -R

# YOLO mode (auto-approve everything)
$ dirge --yolo

09 · FAQ

Frequently Asked

Common questions about dirge.

Q What makes dirge different from other coding agents? ▾
dirge is written in Rust, which means it's dramatically more efficient than JS/TS-based alternatives. ~25MB binary vs hundreds of MB for Node.js-based agents. ~8MB RAM at idle vs ~300MB. It also features a sophisticated session management system with branching, a plugin system in Janet, LSP integration, and a retro TUI built with ratatui.
Q Does dirge support local models? ▾
Yes. Ollama is a first-class provider. You can also use any OpenAI-compatible local server (llama.cpp, vLLM, etc.) via the custom provider config. Set "provider": "ollama" or configure a custom endpoint URL.
Q Which languages does the semantic analysis support? ▾
Tree-sitter grammars are included for TypeScript/TSX, Python, Clojure, Go, Ruby, Rust, Java, C, and C++. Each provides list_symbols, get_symbol_body, find_callers, find_callees, and find_definition. Enable with --features semantic at build time.
Q How does the permission system work? ▾
Four modes: standard (safe ops auto-approved, writes/bash/MCP prompt for approval), restrictive (every tool prompts), accept-all (auto-approve inside cwd, external paths prompt), and yolo (auto-approve everything). Add pattern-based rules like "git checkout *": "allow" for fine-grained control.
Q What is Janet and why is it used for plugins? ▾
Janet is a lightweight, embeddable Lisp-like language (~1MB runtime). It's ideal for plugins because it's fast to start, has a small footprint, and provides a familiar Lisp syntax. Plugins can hook into every lifecycle event — intercept tool calls, modify prompts, register custom commands, and more — without restarting dirge.
